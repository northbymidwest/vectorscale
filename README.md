# VectorScale

RetroArch Slang shader implementing the [Kopf-Lischinski pixel-art vectorization algorithm](https://johanneskopf.de/publications/pixelart/). Converts pixel art into smooth vector boundaries with analytical anti-aliasing, running entirely on the GPU as a multi-pass fragment shader pipeline.

Works with any input resolution — Game Boy, NES, SNES, GBA, or any other pixel art source.

Based on the vectorize-gpu compute shaders in [vibeboy](https://github.com/northbymidwest/vibeboy). That implementation is a cleaner reference for the algorithm since it uses compute shaders directly; this port shoehorns the same pipeline into fragment shaders to work within RetroArch's multi-pass Slang framework.

## Installation

Copy the `retroarch-shaders/` directory into your RetroArch shaders folder, then load `vibeboy-vectorize.slangp` as a shader preset.

Requires a GPU backend that supports Slang shaders (Vulkan, Metal, or Direct3D 12) and `float_framebuffer` (RGBA16F intermediate textures).

## Pipeline

| Pass | Shader | Description |
|------|--------|-------------|
| 0 | similarity-graph | Builds 2x connectivity graph with YUV color similarity thresholds |
| 1 | graph-snapshot | Texel copy for consistent reads during crossing resolution |
| 2 | resolve-crossings | Resolves diagonal crossings via curve length, sparse pixel BFS, and island heuristics |
| 3 | cell-graph | Creates B-spline control points at grid corners with neighbor links, flags, and boundary directions |
| 4 | init-positions | Initializes position offset texture to zero |
| 5-6 | optimize-energy (x2) | Golden section search optimizer minimizing curvature + positional energy |
| 7 | update-tjunction | T-junction position correction (weighted average smoothing) |
| 8 | cell-rasterizer | Renders optimized B-spline curves with multi-hit color resolution and analytical AA |
| 9 | passthrough | Standard texture sample for correct RetroArch viewport mapping |
